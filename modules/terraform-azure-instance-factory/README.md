## 📖 Overview

This Terraform module provisions a single Windows VM into a pre-existing Azure
resource group and subnet, optionally running a PowerShell bootstrap script on
first boot. It is the Azure counterpart to `terraform-aws-instance-factory` and
is intended to be invoked once per lab VM (DC, member server, jumpbox, etc.) by
a higher-level deployment.

The module creates:

- An optional public IP (Standard SKU, static)
- A network interface attached to the caller-supplied subnet, with a dynamic
  private IP and the optional public IP
- A Windows VM (default: Server 2022 Datacenter, Azure Edition) with a
  system-assigned managed identity so it is reachable via Azure Run Command
- An optional Custom Script Extension that executes a caller-supplied
  PowerShell `bootstrap_script` on first boot — equivalent to AWS user-data

The admin password is **not** generated by this module — it is passed in as a
sensitive variable so the calling deployment can keep a single source of truth
across many VMs.

Every resource is tagged with `Environment`, `ManagedBy = "Terraform"`, and
`AccessMethod = "AzureRunCommand"`, plus any caller-supplied `tags`.

## Usage

### Minimal example

```hcl
module "dc01" {
  source = "../../modules/terraform-azure-instance-factory"

  env                 = "staging"
  instance_name       = "dc01"
  resource_group_name = module.network.resource_group_name
  location            = module.network.location
  subnet_id           = module.network.private_subnet_id
  admin_password      = var.admin_password
}
```

### With a bootstrap script and a public IP

```hcl
module "jumpbox" {
  source = "../../modules/terraform-azure-instance-factory"

  env                 = "staging"
  instance_name       = "jumpbox"
  resource_group_name = module.network.resource_group_name
  location            = module.network.location
  subnet_id           = module.network.public_subnet_id
  assign_public_ip    = true
  admin_password      = var.admin_password

  instance_size = "Standard_D4s_v3"

  bootstrap_script = file("${path.module}/scripts/bootstrap-jumpbox.ps1")

  tags = {
    Project = "DreadGOAD"
    Role    = "jumpbox"
  }
}
```

### Notes

- The module does not create a resource group or subnet — pair it with
  `terraform-azure-net` (or any equivalent) to provide those.
- `bootstrap_script` is base64-encoded as UTF-16LE and run via
  `powershell.exe -EncodedCommand`, the standard contract for the Custom
  Script Extension.
- `computer_name` defaults to the first 15 characters of
  `${env}-${instance_name}` (Windows NetBIOS limit). Override it explicitly
  if your name prefix collides.
- `assign_public_ip` defaults to `false`. Outbound internet should normally
  go through the NAT gateway in `terraform-azure-net`; only set this to
  `true` for jumpboxes or other VMs that need inbound reachability.
- The VM gets a system-assigned managed identity so the calling code can
  reach it via Azure Run Command without a separate credential plane.
